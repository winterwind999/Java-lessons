Bridge Pattern
- similar to adapter pattern but works with new code instead

Concepts:
- Decouple Abstraction and Implementation
- Encapsulation, Composition, Inheritance
- Changes in Abstraction won't affect client
- Details won't be right to begin with

Design:
- Interfaces and Abstract classes
- Composition over Inheritance
- More than Composition
- Expect change from both sides
- Abstraction, Implementer, Refined Abstraction, Concrete Implementor

Pitfalls:
- Increases Complexity
- Conceptually difficult to plan
- More than just Object-Oriented
- Confusing What goes Where

Comparison with Adapter Pattern:
- Designed upfront
- Abstraction / Implementation vary
- Built in advance
- Adapts to multiple systems

Summary:
- Design for uncertainty
- Can be complex
- Provides flexibility
- Much more than composition
